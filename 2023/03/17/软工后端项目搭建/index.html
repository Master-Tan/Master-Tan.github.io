docker中的数据库存储：数据卷(弃用)

  • 创建数据卷
    使用以下命令创建一个名为 backend-database-sqlite 的数据卷：
    docker volume create backend-database-sqlite

  • 使用数据卷
    在启动容器时，可以使用 -v 或 —mount 选项将数据卷挂载到容器中。例如，以下命令将创建一个名为 mycontainer 的容器，并将数据卷 mydata 挂载到容器的 /data 目录：
    docker run -d —name mycontainer -v mydata:/data myimage
    在容器中，可以在 /data 目录中读写数据，这些数据将存储在 mydata 数据卷中。当容器被删除时，数据卷将仍然存在，并可以挂载到其他容器中。

  • 查看数据卷
    使用以下命令可以列出所有的数据卷：
    docker volume ls
    使用以下命令可以查看特定数据卷的详细信息：
    docker volume inspect mydata
    在输出中，可以找到数据卷的名称、挂载点、权限等信息。

  • 删除数据卷
    使用以下命令可以删除一个未被挂载到容器中的数据卷：
    docker volume rm mydata
    如果数据卷已经被挂载到容器中，需要先将容器删除，然后再删除数据卷。例如：
    docker rm mycontainer
    docker volume rm mydata
