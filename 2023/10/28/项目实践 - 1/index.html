metadata.json

  • “bounds”: 表示数据的空间边界，它是一个二维数组，包含了两个坐标点，分别表示数据的最小和最大边界。

  • “sequence_length”: 表示序列的长度，它指定了数据中每个序列的时间步数。

  • “default_connectivity_radius”: 表示默认的连接半径，用于定义数据中的连接性。具体含义可能取决于上下文，可能用于定义数据中的邻近关系或相似性。

  • “dim”: 表示数据的维度，这里是二维数据。

  • “dt”: 表示时间步长，表示数据中每个时间步的时间间隔。

  • “vel_mean”: 表示速度的均值，它是一个二维数组，包含了两个分量的均值值。

  • “vel_std”: 表示速度的标准差，也是一个二维数组，包含了两个分量的标准差值。

  • “acc_mean”: 表示加速度的均值，同样是一个二维数组，包含了两个分量的均值值。

  • “acc_std”: 表示加速度的标准差，同样是一个二维数组，包含了两个分量的标准差值。

处理数据 extract_tfrs.py -> data/(test、train、vaild)/.pkl

“extract_tfrs.py” （extract tensorflow records.py）的主要作用是从 TensorFlow 记录（TensorFlow Records）文件中提取数据。

TensorFlow Records 是一种用于存储大规模数据集的二进制文件格式。它可以有效地存储和读取大量训练数据，提供高性能的数据加载能力，特别适用于使用 TensorFlow 进行深度学习训练。

“extract_tfrs.py” 是一个自定义的 Python 脚本，其目的是从 TensorFlow Records 文件中解析和提取数据。这个脚本可能会使用 TensorFlow 或其他相关的库来读取和处理 TensorFlow Records 文件，并将数据转换为适合特定任务的格式。

animation.FuncAnimation

https://matplotlib.org/stable/api/_as_gen/matplotlib.animation.FuncAnimation.html

animation.FuncAnimation 是一个用于创建动画的函数。下面是对参数的解释：

fig: 这是一个 matplotlib 的图形对象，指定了动画将在哪个图形上进行绘制。
update_points: 这是一个函数，用于在每个动画帧中更新图形的点的位置。
frames: 这是动画的总帧数，控制动画的长度。
fargs: 这是传递给 update_points 函数的额外参数。它们是 pts1、pts2、tgt_pos_seq 和 pred_pos_seq，用于更新图形中的点位置。
动画对象 ani 是通过调用 animation.FuncAnimation 创建的。创建动画后，可以将其用于显示或保存动画效果

变量命名

tgt_pos_seq: 目标位置序列（Target Position Sequence）
pred_pos_seq: 预测位置序列（Predicted Position Sequence）

TORCH.UTILS.DATA

官方文档：https://pytorch.org/docs/stable/data.html (英文)
https://www.pytorchtutorial.com/docs/package_references/data/ (中文)

更好理解的版本：https://blog.csdn.net/zhangjipinggom/article/details/100310823

在代码中，使用 for batch_idx, data in enumerate(self.data_loader): 这样的循环遍历 data_loader 中的数据时，data 的维度通常会增加一维。这是因为 data_loader 会返回一个批次的数据，而每个批次的数据通常是一个多维数组或张量。

如果原始数据的形状是 (…, dim1, dim2, …)，那么在遍历 data_loader 后，data 的形状将变为 (batch_size, …, dim1, dim2, …) 。也就是说，原始数据的维度保持不变，但新添加了一个维度来表示批次。

这是为了方便批处理操作和并行计算。在许多深度学习框架中，数据通常以批次的形式传递给模型进行训练或推理。通过在数据加载过程中添加批次维度，可以更方便地进行批处理操作和并行计算，从而提高训练和推理的效率。
