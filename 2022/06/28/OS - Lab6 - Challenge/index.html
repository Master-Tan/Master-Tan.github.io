Easy 部分

(1) 实现后台运行

  • 实现思路：

    • 目前的 shell 在等待输入时，是在内核态忙等

    • 后台运行：fork 出子进程，在子进程等待此命令结束，父进程（主进程）则继续执行 Shell 程序

    • 因此，增加一个后台运行标记变量 handup，在读取到 '&' 时置高，根据此变量来选择是内核态忙等还是 fork 出子进程使其在子进程运行来实现 后台运行 的效果

总结

一开始拿到这三个任务的时候，还犹豫挺久应该选择哪一个，最后还是选择了相对而言更加熟悉的 Lab6 ； Lab6 的 Challenge 能通过敲自己熟悉的 Linux 命令就能展示结果，题目更加通俗易通，原理相比也比较简单，可以通过一点一滴的反馈逐步实现较为复杂的功能；整个 Challenge 完成下来花费了大量的时间精力，其中主要的问题是功能的设计与实现，挺多东西下手去做就感觉举步维艰，最后算是很艰难的压线做完了这个挑战性任务；但是收获也是与付出成正比的：笔我于实验的操作系统 MOS 的整体性理解可以说更上一层楼；通过完善我的 Tan's Shell ，我们的操作系统不再那么冰冷，而是可以进行简单的和甚至稍稍复杂一点与使用者交互；结合 CO 来看，某种程度上说，我们已经在理论上可以从逻辑门开始，直至完成一个具有交互能力的现代计算机了
