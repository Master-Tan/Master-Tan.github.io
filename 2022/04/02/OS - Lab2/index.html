Thinking 2.3

请阅读 include/queue.h 以及 include/pmap.h, 将 Page_list 的结构梳理清楚，选择正确的展开结构

答：

  • MIPS 系统加电起动后， MIPS 处理器默认的程序入口是 0xBFC00000 ，此地址在无缓存的 KSEG1 的地址区域内，对应的物理地址是 0x1FC00000 ，即 CPU 从 0x1FC00000 开始取第一条指令，这个地址在硬件上已经确定为 FLASH 的位置，然后 Bootloader 执行功能，加载内核到内存（ Linux 内核镜像文件拷贝到 RAM 中某个空闲地址（ LOAD 地址）处），之后在本系统中将 ./tools/scse0_3.lds 传给链接器 ld ， ld 会将 .text 节的地址链接到 0x80010000 （即内存布局图中内核代码开始处）。关于内核 ELF 文件的入口地址 (Entry point) ，即 bootloader 移动完内核后，直接跳转到的地址，由 ld 写入 ELF 的头中，在本系统中通过 ENTRY(_start) 指明并跳转至内核启动代码 ./boot/start.S 处

      使状态寄存器 SR (Status Register) 全部位置 0

      禁用了中断

  • mfc0 t0, $16
    and t0, ~0x7

    mtc0 t0, $16

    • 读/写寄存器：
      取出 CP0 第16号寄存器 Config（CPU参数设置寄存器）,并将后三位置 0，再将第二位置 1，并写回 Config 寄存器

    • 达到目的：
      禁用了内核模式缓存（Cache）

lab1-2-exam:

题目是给出一个包含一个 int ，一个 char ，一个 int[SIZE_C] 数组（其中 SIZE_C 是可变的常量）的结构体，然后给定标识符 T ，让你用类似 {0001, a, -001, 0002} 的方式在 printf 的函数中用类似 printf("%T", &my_struct); 的方式输出该结构体
我的上机体验极差，一开始就想歪了，在本地定义了一个结构体，数组我还定义为了 int* 的指针来企图取到数组的首地址，然后就不知道啥原因取到了一些奇奇怪怪的地址导致输出乱七八糟，考试时间延长了半小时（估计通过率不够）也没做出来，纯纯坐大牢。
本题正确思路是（后来问助教）先用 va_arg(ap, int *) 取到结构体中 int 的首地址，然后偏移一个 int 的距离即为 char 的地址， 再偏移一个 int 就是数组第 0 个的地址…以此类推，再按样式分别输出即可
