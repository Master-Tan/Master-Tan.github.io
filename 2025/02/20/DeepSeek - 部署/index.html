模型部署位置

如果您按照自述文件所述以自动方式安装了 ollama：
打开 systemd 文件

在[Service]下添加 Environment=”OLLAMA_MODELS=/your/desired/path”

按CTRL+X然后按Y然后按 Enter

或者：
您所要做的就是在运行 ollama 之前设置环境变量 OLLAMA_MODELS。

GPU 选择

如果您的系统中有多块 NVIDIA GPU，并且想要限制 Ollama 使用子集，则可以将其设置CUDA_VISIBLE_DEVICES为以逗号分隔的 GPU 列表。可以使用数字 ID，但顺序可能会有所不同，因此 UUID 更可靠。您可以通过运行来发现 GPU 的 UUID。nvidia-smi -L如果您想忽略 GPU 并强制使用 CPU，请使用无效的 GPU ID（例如“-1”）

ssh root@10.3.3.107 -p 5233

export OLLAMA_MODELS=/workspace/ollama/models
export CUDA_VISIBLE_DEVICES=GPU-dd8aa6f3-b29f-6158-51bf-ddb94b6954ea,GPU-10caa885-bf51-4c17-a876-1d14a8928f15,GPU-6b47013b-03b0-b397-9449-c59e78c1bb16
nohup ollama serve > /workspace/ollama.log 2>&1 &

“-d”: 后台运行

后台运行

安装 screen（如果容器中没有）：
apt update && apt install screen

启动一个新的 screen 会话：
screen -S dev-session

在 screen 会话中运行 npm run dev：
npm run dev

按下 Ctrl+A，然后按 D 分离会话。即使终端关闭，进程也会继续运行。

重新连接到会话：
screen -r dev-session

screen -S ollama-session
export OLLAMA_MODELS=/workspace/ollama/models
export CUDA_VISIBLE_DEVICES=GPU-dd8aa6f3-b29f-6158-51bf-ddb94b6954ea,GPU-10caa885-bf51-4c17-a876-1d14a8928f15,GPU-6b47013b-03b0-b397-9449-c59e78c1bb16
ollama serve
按下 Ctrl+A，然后按 D 分离会话。即使终端关闭，进程也会继续运行。
重新连接到会话
screen -r ollama-session

删除会话
screen -X -S ollama-session quit
